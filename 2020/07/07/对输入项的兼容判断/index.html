对属性是否存在的判断、对输入内容的兼容

2020-07-07
我常犯的错误

问题：

1.

当对象为空，第二个attributes就会报错

可以参考：https://blog.csdn.net/weixin_34200628/article/details/88609409

  1. 用户未输入时，会默认传’undefined’,不管应该传什么，传undefined，肯定是不对的，需与后端确认未输入情况应传默认参数值，或是不传。

综合解决方案

  1. 第一步：对接口的确认，不管是输入项还是返回项，确认是 required 还是 optional 的
  2. 针对具体的 field / object 来加上 assertion 断言(not null, not undefined)，过滤，或者是出错提示
  3. 对于数据结构的处理方法，因为是独立于界面层的逻辑，所以可以单独抽出来文件(data model)，这样边界条件，空与否的判断可以加到 unit test 中。发版前确保这些 tests 都通过。涉及到的依赖会有 mocha, chai（？学习学习待学习）
  4. 对于不同于数据处理的情况，比如界面逻辑，也可以整体写 ui integration tests。一般用的依赖是 react-test-renderer, jest
  5. 最后最直接也是不可缺少：对于核心的功能在测试环境多测多测多测 再发到线上
